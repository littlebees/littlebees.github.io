git rebase 與 stash

Posted on 2020-09-30 Edited on 2022-04-13 In Utils , Git

動機

之前上code，因為各種因素，修與加了許多分散的功能

都分散在不同的小commit中，但總不能在code review與push時上一堆零散的commit

所以有了這篇

rebase

可以讓你從起點commit開始往下修commit
可以合併與分割commit

合併很簡單，所以這邊特別說分割commit的手法

分割commit

  1. 把檔案從commit拉回來: git reset HEAD^
  2. 照常add與commit

但如果只想commit其中一些檔案的某些部份?
就要設法把untracked狀態保留下來，以及只選擇性的stage部分內容

先談untracked狀態保留

保留untracked&staged: stash

可以利用stash把untracked&staged狀態存下來，
等之後的需要再pop出來，就是commit之前的狀態

但這裡是會把staged都保留下來，但我們只要untracked的保留

所以要

  1. 只存staged: git stash -u
  2. 存untracked&staged: git stash -k OR git stash --keep-index

之後就是pop了

再來就是選擇性的stage部分內容

選擇性的stage部分內容

git add --patch <yourfile> OR git add -p <yourfile>

在stage的過程中可以用

  • y stage this hunk for the next commit
  • n do not stage this hunk for the next commit
  • q quit; do not stage this hunk or any of the remaining hunks
  • j leave this hunk undecided, see next undecided hunk
  • k leave this hunk undecided, see previous undecided hunk
  • s split the current hunk into smaller hunks
  • e manually edit the current hunk
