一個面試問題

Posted on 2020-10-17 Edited on 2022-03-16 In Algorithm

動機

面試遇到的問題，當下沒想到DP的解法，但考官好像也沒有往DP的方向提示(還是因為時間不夠…)
但題目確實可以用DP解，所以就記錄下來

問題

類似子字串，字串的token用空格分開，看token是不是都存在在字串中
“a b x c x”.isLikeSubStr(“a b c”) => Yes

“a x c x”.isLikeSubStr(“a b c”) => No

Sol ver.1 & Sol ver1.5 : Hash

第1版是把兩個字串轉成hash table統計數量，再比較數量

節省空間的版本，第1.5版是只轉第一個字串，之後把第二個字串的token往hash table裡面扣

變種: 順序

現在多了要照順序這要求
“a c x b x”.isLikeSubStr(“a b c”) => No (要照順序)

這時候就是DP的時候，照順序就是遞迴的強烈暗示，或是說最佳子結構

列出兩條所有起點的可能性，就是一張len(strA)*len(strB)的表

實際步驟如下

  1. 從strB的第一個開始，所有等於第一個的token都標上true

  2. 之後就是從有True的點從上一排開始往左邊找有沒有True

  3. 只要最後的row有true就是true

當前的結果都是只用上一層的答案，所以可以只用兩條array就完成dp

空間複雜: len(strA)
時間複雜: len(strA)*len(strB)

DP 與 Hash的差異

DP可以記下順序，但hash不行
不過hash因為少了順序的限制，其時間複雜就是len(strA)，不是DP的len(strA)*len(strB)
