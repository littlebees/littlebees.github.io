graphql的理解

Posted on 2021-10-31 Edited on 2022-03-31 In Web , Rails , FAQ

動機

有碰到，紀錄一下

基本上就是

query

query可以當成，

  • 一個function call，後面接filter
  • 在filter中依據前面function call的return，再取值做function call
  • 直到遇到return值是scalar type，如果還是function就繼續call

server

  • 定義三顆tree，query, mutation, subscription
    • tree的node就是物件的attribute
      • attribute對到function或是scalar type
        • 所以不能直接回array之類的collection，這裡是graphql會處理

例子

schema定義tree的樣子與type

resolver定義怎麼抓值，重要的是function的signatrue

  • parent就是上一層的回傳值
  • args就是定義在schema的參數
  • context在定義graphql server時給定的全域變數

順便嘴一下js的function，可以省略參數是怎麼回事
這樣很難讀懂，也不好找資料，因為根本沒關鍵字，
另外函數最重要的就是參數，連這個都不知道有什麼是要怎麼開發，js這個設計真的不行

query就是query

呼應到attribute就是function，注意到只能用named arg去call!!

用function與filter來看

整個query也可以當成function

query還有其他工具，但不是重點，跳過

subscribe

這邊沒有去實驗subscribe，但可以猜一下是幹嘛

subscribe就是連線，負責拿publish的資料
resolve負責對收到的資料做修飾，在傳回去user之前
