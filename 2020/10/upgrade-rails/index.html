升級rails

Posted on 2020-10-22 Edited on 2022-03-31 In Rails , Tips

動機

把以前的專案升級，不然這作品不能看…

步驟

  1. 改Gemfile，把所有版本號砍掉，只留下rails
  2. bundle update
  3. THOR_MERGE=“diff” rails app:update
  4. 跑測試，確保每一頁都能動qqqq(自動測試的重要qqqq)

下面就是在每個升級的階段遇到的問題

rails 4 to rails 5

1. Please specify merge tool to THOR_MERGE env.

指定merge的指令就好，想一勞永逸

2. Directly inheriting from ActiveRecord::Migration is not supported

錯誤訊息大概像這樣

解法，把migration換掉

3. Array values in the parameter to Gem.paths= are deprecated.

雖然說是警告，解法

4. Sprockets::FileNotFound - couldn’t find file ‘base.source.coffee’

這應該算是臨時解法，但之後會換成webpack所以先這樣吧

在config/environments/development.rb
把asset compile關掉

5. Expected to find a manifest file in app/assets/config/manifest.js (Sprockets::Railtie::ManifestNeededError)”

當初處理這個是直接生檔案，這樣就不會報錯，但這個問題好像與上一個有關…

6. Undefined method raise_in_transactional_callbacks=’ for ActiveRecord::Base:Class (NoMethodError)

把config/application.rb的

註解掉

7. rails generate devise:install 卡住

8. Couldn’t find Admin::BaseHelper, expected it to be defined in helpers/admin/base_helper.rb

明明就在…

解法
在 config/development.rb 設定

9. NoMethodError: undefined method new for BigDecimal:Class

Ruby不能用超過2.5

rails 5 to rails 6

1. Resource must exist

這是Rails5之後，要求所有belong_to必須是require
也就是外鍵必須存在

所以把 app/model/role.rb 改一下

其實如果跑rolify install 就會有新的檔案，但當初我選合併…
所以差不多的code出現兩次，進而出現很奇怪的結果(this title)

2. 改繼承ApplicationRecord

自rails5後，model都是從ApplicationRecord繼承，所以要把ActiveRecord::Base換掉

新增 app/models/application_record.rb

把所有ActiveRecord::Base換成ApplicationRecord

3. load_defaults

改config/application.rb，升級成rails6

4. User#index not exists

這是升級devise的鍋，我也不知道為什麼會出事??

加上需要的controller就好

為什麼不先寫測試

以前當學生東西會動就好，根本不會想到要加測試…

像升級 或是 改、加功能只要測試跑下去就好，多輕鬆

下一次就是加測試，但測試是要測試什麼感覺要研究一下
像升級就是會錯在一些感覺不會錯的地方(不是自己程式邏輯的錯)，像是

  1. api的signature改變
  2. 新rails多新code
  3. Class改名字
  4. deprecated method

心得

  1. 善用merge tool (好好看完再merge)
  2. 寫測試
  3. 小心警慎
