rendezvous hash與consistent hash

Posted on 2022-03-14 Edited on 2022-03-31 In Algorithm , System , Distributed

動機

先假設

我們有

  • srv2key: 把server對到負責的key們
  • key2srv: key是哪一個srv負責
  • srvs: 所有server

consistent hash

先想像有一個hash可以把string轉成0~2**32-1之間的值

把servers想像成環上的點，之後分配資料就是利用bsearch找靠近的server

刪server時，只要往把原本的server資料往前丟就好

rendezvous hash

可以把consistent hash擴展成把所有server加進來比，選一個分數最高的

刪server時，就變成每個key都要重新map

rendezvous hash & consistent hash

rendezvous hash根據hash function，可以讓key平均分配在server上
但是如果要add/del server是個大工程。

consistent hash在處理add/del server簡單的多，同時查找是log n不是n。
但是如果有hotspot，就要自己調整virtual node去調。
