oauth2、openid、jwt筆記

Posted on 2021-11-04 Edited on 2022-07-30 In Web , Rails , FAQ

動機

記錄用

授權 & 驗證

驗證(authentication): 確認身分 (openid)
授權(Authorization): 可以使用某個資源 (oauth2)

oauth2

有4種流程
authorization code流程是

  1. 使用者(resource owner)透過browser(user-agent)連到app(client)
  2. app redirect到google的server(Authorization Server)
  3. google登入、approve可以用這方面的資源
  4. google給redirect url與authorication code
  5. browser做redirect，讓app收到authorication code
  6. app再用authorication code去換access token!!
  7. 用access token拿google上的資源(Resource Server)

整個最重要的是

  1. 拿authorication code
  2. 用authorication code換access token

還有其他(簡化)流程，像

  1. (Resource Owner Password Credentials Grant) 把帳密給app讓他直接與Authorization Server拿access token
  2. (Client Credentials Grant) 只有app與Authorization Server，其實就是一般的登入
  3. (implicit grant type) 剩下一種是讓browser把authorication code自己在換出access token，之後給app

openid

openid是驗證身分，流程類似oauth2但是最後傳的東西不只是資源，還有對面的(google的)用戶識別資訊，簡單來說就是把別人的user當成自己的user用

oath2是授權，所以不同server的資源都不一樣，導致scope都不一樣
但openid是驗證，所以user權限(scope)會是通用的

整個流程與authorization code流程很像，但是最後傳的是id token，以及server的user識別訊息

另外提Single Sign On (SSO)，就是openid完成的事，透過其他server完成使用者驗證

jwt

就是token的格式

token有user的資訊，就可以不用session另外map到user去
format是標頭.內容.簽名

標頭: 加密algo, token種類
內容: json data
簽名: 把標頭.內容用私鑰加密的結果

或是反著用，用公鑰加密，但是用私鑰解密，但是會變得很複雜，可以看這邊
