css的context

Posted on 2022-01-27 Edited on 2022-03-12 In CSS , FAQ

動機

寫個簡單的筆記

tree的node: box(element)

box就是每個element，有各自的width, margin, padding

但有block與inline讓margin的計算方式不同

tree root: Formatting Context

決定裡面的node怎麼顯示在畫面上排版

block Formatting Context

最一般的FC，就是把block element從上往下一直塞
inline塞成一行，這一行當成block

產生條件

  • 根元素或其它包含它的元素
  • 浮動 float: left/right/inherit
  • 絕對定位元素 position: absolute/fixed
  • 行內塊 display: inline-block
  • 表格單元格 display: table-cell
  • 表格標題 display: table-caption
  • 溢出元素 overflow: hidden/scroll/auto/inherit
  • 彈性盒子 display: flex/inline-flex
    • Flex Formatting Contexts
  • grid: display: grid
    • Grid Formatting Contexts

性質

因為這是tree的根，所以會有一些有趣的性質

Margin Collapsing

在同一個BFC下，會把下面的box的top-margin吃掉

發生在

  • 第一個與最後一個child
  • 兄弟

其實就是讓margin盡量靠近當初設定的

如果在同一個context下，都還保持各自的margin，這樣margin會超長，所以可以取一個做代表就好

reflow

如果dom被動過(js之類的)，這樣要重新排版，這叫reflow

但只有在同一棵tree下在的才會被影響(同一個context)，所以可以利用生不同context把reflow的範圍變小，加快reflow的速度

visibility & display
  • visibility
    • 看不看的到
      • 會被reflow，但不會被repaint
  • display
    • 決定要怎麼排版(Formatting Contexts)
      • display: none就是連reflow都不會

Inline Formatting Contexts

可以算是Block Formatting Contexts的特例

就Block Formatting Contexts只有inline element

stacking context

前面看過了兩種tree，主要是對平面做排版

那三維就是每個tree的平面中切出好幾個level，之後就可以把tree的node指定到各個level去

這個level叫z-index

如果都在同一個level
